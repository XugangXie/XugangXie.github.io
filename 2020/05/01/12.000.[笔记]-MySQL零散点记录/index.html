创建数据库

create database xxx default character set utf8 collate utf8_general_ci;

修改 mysql 不区分大小写

找到配置文件, 在 [mysqld] 下增加 lower_case_table_names=1

设置不使用 ONLY_FULL_GROUP_BY 模式

在配置文件中 [mysqld] 下增加
sql_mode=STRICT_TRANS_TABLES,NO_ZERO_IN_DATE,NO_ZERO_DATE,ERROR_FOR_DIVISION_BY_ZERO,NO_AUTO_CREATE_USER,NO_ENGINE_SUBSTITUTION

修改时区

在配置文件中 [mysqld] 下增加
default-time_zone = '+8:00'

不使用缓存查询

select sql_no_cache * from xxx

INSERT INTO person_info (username, age) VALUES ('lin', 22),('bin', 21)

UPDATE person_info SET age=23 WHERE username='bin'

ALTER TABLE person_info ADD UNIQUE INDEX idx_id (id)

排序规则

utf8_general_ci 中 ci 表示 case insensitive, 不区分大小写排序。
utf8_bin 表示字节排序，会区分大小写。

行长度限制

行长度最大是 65535 字节， 如果是 utf8 编码， varchar 的长度是 65535 减去其他字节数再除以 3.

复制一个表

varchar 字段使用整型查询

假设有表 t_test, 有字段 row_id varchar(10), 这个字段上有建立索引, 表数据如下:

row_id
1
1a

执行查询: select * from t_table where row_id=1, 结果是不会走索引的, 并且把 1a 这条记录也查出来.
如果执行: select * from t_table where row_id='1', 这样就走索引了, 并且不会查出 1a 这条记录.
原因是因为 mysql 对于不用类型的参数, 会把它们都转成浮点数再来比较, 这样就不能走索引了, 只能全表扫描, 而且 1a 转成浮点数是 1, 所以会查出它.

如何保证事务原子性

使用 undo log, 在操作数据之前把数据先备份到undo log, 当需要回滚时, 把undo log的数据恢复即可.

如何保证事务持久化

数据库是把要修改的数据所在的数据页先读到内存, 然后修改数据再写入数据库. 为了保证持久化, 可以使用redo log, 把修改的数据先写到redo log, 当事务提交时再把redo log写进数据库即可.

如何保证事务的隔离性

InnoDB 引擎使用 mvcc（多版本并发控制） 的方式来读取数据，这个方式会让每一行都会创建两个隐藏字段，一个是表示这条数据的创建事务版本号DB_TRX_ID；一个是回滚指针DB_ROLL_PTR，用来指向undo log 的记录，这样会形成一条链，可以找到修改之前的数据。
要注意的是update操作实际是增加一条记录，然后把原来记录标志为deleted。
InnoDB 默认是可重复读，在一个事务中使用当前读取一条数据，实际就是看这条数据的DB_TRX_ID是不是对当前事务可见，不可见就通过DB_ROLL_PTR找到可见的数据。
通过这种版本号控制的方式就可以很好的隔离并发事务的操作了，而且不需要加锁。另外在 InnoDB 中这种方式也可以很好解决了可重复读的幻读问题。

联合索引

假设联合索引: (a, b, c)
那么下面可以直接通过联合索引得到结果：
select * from table where a=xxx and b=xxx order by c

但是下面这样就不行, 还需要执行一次filesort排序:
select ... from table where a=xxx order by c
因为没有 (a, c) 的联合索引.

索引和锁

如果查询条件是索引字段, 那么会锁住索引及对应范围;
如果查询字段没有建立索引, 那么会全表扫描导致全表锁住.

删除大表数据

为了防止使用delete操作导致锁表, 需要下面操作:

备份不需要删除的数据
insert into em_event_extend_new select * from em_event_extent where row_id>265850266
修改表名
rename table em_event_extend to em_event_extend_old;
rename table em_event_extend_new to em_event_extent;
删除旧表
drop table em_event_extend_old;

using index

在索引中查询指定值. 例子:
SELECT sys_created_on from em_alert where sys_created_on='2019-07-22'

using where; using index
在索引中查询范围值或者需要判断索引字段的具体值, 就是条件过滤索引字段. 例子:
SELECT sys_created_on from em_alert where sys_created_on>='2019-07-22'

using index condition
条件过滤索引字段, 返回数据行, 再用 where 其他过滤具体行. 例子:
SELECT sys_created_on from em_alert where sys_created_on>='2019-07-22' AND description='111'
