几种锁类型

InnoDB中，乐观锁、悲观锁、共享锁、排它锁、行锁、表锁、死锁

意向共享锁（intention shared lock, IS）表示事务打算在表中的各个行上设置共享锁。
意向排他锁（intention exclusive lock, IX）表示事务打算在表中的各个行上设置排他锁。
SELECT … LOCK IN SHARE MODE 设置IS锁， SELECT … FOR UPDATE设置IX锁

数据库中的锁类型

表锁 冲突概率高于行锁
行锁 并发性能高于表锁（冲突概率越低，并发性能越高）

读锁
加锁方式 select * from table LOCK IN SHARE MODE;
写锁

从锁的粒度角度：
R锁-共享锁
加锁方式 select * from table LOCK IN SHARE MODE;
释放方式 commit/rollback
X锁-排他锁
加锁方式 自动 delete/update/insert
手动 select * from table FOR UPDATE;

悲观锁
乐观锁

从锁的算法角度:
IX 意向排它锁
? 插入意向锁
自增锁
意向锁
临建锁 Next-Key Lock
间隙锁 Gap Lock 是专门用来阻塞插入的
谓词锁

记录锁： Record Lock 使用唯一性索引做等值查询，精确匹配的时候，会锁住一条记录
