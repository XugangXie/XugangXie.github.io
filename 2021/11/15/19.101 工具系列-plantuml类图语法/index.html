UML类图中的几个箭头含义：

  • **1.泛化关系(generalization)**：继承关系的一种表现形式，箭头指向被继承的父类，这个父类不是抽象类。UML语法：AbstractClass <|-- Class

  • **2.实现关系(realize)**：继承关系的一种表现形式，箭头指向被继承的父类，这个父类是抽象类。

    实现关系用一条带空心箭头的虚线表示；UML语法：Interface <|.. Class

  • **3.聚合关系(aggregation)**：

    聚合关系用一条带空心菱形箭头的直线表示，如下图表示A聚合到B上，或者说B由A组成；UML语法：B o--> A 或 A <--o B

  • **4.组合关系(composition)**：

    组合关系用一条带实心菱形箭头直线表示，如下图表示A组成B，或者B由A组成；UML语法：B *--> A 或 A <--* B

  • **5.关联关系(association)**：UML语法：--

  • **6.有方向的关联关系(directed association)**：

    关联关系默认不强调方向，表示对象间相互知道；如果特别强调方向，如下图，表示A知道B，但 B不知道A；UML语法：A --> B

    注：在最终代码中，关联对象通常是以成员变量的形式实现的；

  • **7.依赖关系(dependency)**：UML语法：A ..> B

聚合 和 组合 的区别：

  • 现在更多的是强调继承，好像处处都想到继承，其实，组合更加的灵活，应首先考虑组合。

  • 聚合关系,当A创建的时候，B不一定创建；当A消亡时，B不一定消亡。

  • 组合关系类图表示：

plantuml语法：

  • 组合与聚合的关系：组合和聚合是有很大区别的，这个区别不是在形式上，而是在本质上：
    比如A类中包含B类的一个引用b，当A类的一个对象消亡时，b这个引用所指向的对象也同时消亡（没有任何一个引用指向它，成了垃圾对象），这种情况叫做组合，反之b所指向的对象还会有另外的引用指向它，这种情况叫聚合。

is a / has a / like a

  • 继承关系即继承关系即 is a 关系，子类继承父类的属性方法；比如：我 is a 人；再比如菱形、圆形和方形都是形状的一种，那么他们都应该从形状类继承而不是聚合/组合关系。
  • 聚合关系即 has a 关系，两个对象之间是整体和部分的关系;比如:我 has a 头;再比如电脑是由显示器、CPU、硬盘这些类聚合成电脑类,而不是从电脑类继承。
  • 组合关系即 like a 关系；

plantuml类图语法

  • 接口、抽象类、实现类的继承、实现
  • 实线+空心三角: 继承（泛化generalization） 语法: <|-- 或 --|>
  • 虚线+空心三角：实现（实现realize） 语法: <|.. 或 ..|>
  • 实线+空心菱形：聚合（aggregation）语法：o-- 或 --o
  • 实线+实心菱形：组合（composition）语法：*-- 或 --**
  • 实线：关联(association)语法：--
  • 有方向的实线: 有方向的关联(directed association)语法：<-- 或 -->
  • 有方向的虚线：依赖(dependency)语法：<.. 或 ..>
