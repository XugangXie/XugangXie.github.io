这篇主要演示，如何将自己构建的镜像，推送到docker私服
docker私服，可以是公司搭建的，也可以是公共的docker hub
以下都是在win10 pro上操作

演示将nginx镜像推送到公司私有镜像库：

环境：

  1. 本地构建镜像的目录: E:\dev\docker-images\fanyi-hub
  2. docker registry私服仓库(公司内部CSP私有云平台-基于Rancher)

前提条件

  1. gitlab：存放项目temp-nginx；
  2. docker registry私服仓库(公司内部CSP私有云平台-基于Rancher)；

构建与推送步骤：

  1. 到本地源码工程目录下:D:\project\temp-nginx，添加dist目录和Dockerfile文件
  • dist这个目录里面放的是index.html和一些静态资源文件；
  • Dockerfile文件，内容如下：

注：该项目托管在gitlab仓库：

  1. 在目录下:E:\dev\docker-images\fanyi-hub，构建镜像：

语法：docker build -t {repo}:{tag} {Dockerfile所在相对目录}
其实 build 完整的语法是:docker build -t {user}/{repo}:{tag} {Dockerfile所在相对目录}

  1. 继续在目录下：E:\dev\docker-images\fanyi-hub，将本地镜像打个tag:

语法：docker tag {本地已经build过的repo}:{本地已经build过的tag} REGISTRYHOST/{租户USEERNAME}/{仓库repo}:{new-tag}

  1. 登录到 REGISTRYHOST: registry.sxc.sh:

  2. 已经打过tag的镜像，推送到 REGISTRYHOST: registry.sxc.sh:

    说明下：registry.sxc.sh/tspdr/temp-nginx:v1180
    registry.sxc.sh: 远端的REGISTRYHOST
    tspdr: 用户名userName
    temp-nginx: 仓库名repo
    v1180: tag

  1. 以上就已经将打过tag的镜像推送到了docker registry私服仓库
