  • 本篇将将演示如何将spring-framework源码导入spring tool suit中

环境说明

  • 操作系统：windows 10 专业版 64bit
  • spring-framework: tag: v4.3.0.RELEASE
  • spring tool suit: 3.9.0，需要事先安装好groovy插件
  • JDK: 1.8.0_131
  • Git: 2.18.0.windows.1

导入流程：

  1. spring-framwork源码托管在github上：https://github.com/spring-projects/spring-framework/commits/v4.3.0.RELEASE， 使用git clone 命令，将源代码拉取到本地：

  2. clone下来的默认是master分支，目标是研究源码:v4.3.0.RELEASE版本，spring-framework发布版本，都是tag，因此看下tag情况：


    发现tag太多，使用正则过滤查询：


    可以发现tag里有 v4.3.0.RELEASE

  3. 基于tag: v4.3.0.RELEASE创建branch: v4.3.0.RELEASE :

  4. 进入目录spring-framework，双击执行import-into-eclipse.bat

有时会莫名的退出，如果这样，再次双击执行import-into-eclipse.bat
当出现BUILD SUCCESSFUL的提示时，表示构建成功，按照它的提示，导入到STS

  1. 导入到STS:



讲过漫长的STS构建后，左边的项目模块仍有一个报错：


点击一个错误，发现是groovy脚本中，缺少导包路径语法错误提示所致，这个应该是该分支里的语法错误，回到github上的master分支查看比较下：


确实是由于这个分支的这三个groovy脚本有语法错误，在这三个报错的脚本上加上导包语句：package org.springframework.scripting.groovy;

至此，spring-framework源代码项目都已经成功导入STS了.
